What is Decoy Mode?

Decoy mode hides two payloads in the same carrier — a fake "decoy" and your real secret. Under pressure, you can reveal the decoy key, which returns innocent content. Only you know the real key that unlocks the actual payload. This is called plausible deniability.

Detect Hidden Content

Analyze a suspicious file for signs of steganography using multiple statistical and forensic methods.

🔬
Seven detection engines:
  • Chi-Square: Detects LSB manipulation by analyzing pixel value distribution — natural images don't have equal pairs, hidden data does.
  • RS Analysis: Estimates what percentage of pixels carry hidden data without needing the key.
  • EXIF/Metadata: Scans all metadata fields, comments, thumbnails, and custom keys for forensic anomalies.
  • Blind Extractor: Tries all common encoding parameters (channels, depths, orders) and scores what it finds.
  • ML Steganalysis: Uses ONNX neural networks to identify deep adaptive steganography that defeats traditional statistical attacks.
  • Camera Fingerprint: Analyzes the PRNU (sensor noise) of real photos; steganography irreparably disrupts this unique fingerprint.
  • Binary Analysis: Checks software binaries (PE/ELF/etc) for structural slack space, EOF attachments, and unnatural entropy spikes.

Why check capacity first?

Each carrier has a maximum payload size depending on its dimensions, bit depth, and encoding method. A 512×512 PNG at depth=1 can hold about 96 KB. A 3-second WAV file can hold about 33 KB. If your payload (after encryption) exceeds this, encoding will fail — check capacity first.

How to Use StegoForge

A beginner-friendly guide to steganography and every feature in this toolkit.

🤔

What is Steganography?

Steganography is the practice of hiding secret information inside ordinary files — unlike encryption, which makes data unreadable, steganography makes the data undetectable. A photo with a hidden message looks exactly like a normal photo.

StegoForge combines steganography AND encryption: your payload is first encrypted (AES-256) then embedded, so even if someone finds the hidden data, they can't read it without your key.

🖼

Image Methods

  • LSB (Pixel): Changes the last 1–4 bits of each pixel's color value. A 255-blue pixel becomes 254-blue — completely invisible. Best choice for PNG and BMP.
  • DCT (JPEG): Embeds data in the math behind JPEG compression (frequency coefficients). More resistant to statistical detection than LSB.
  • Alpha Channel: Hides in the transparency value of PNG/WebP pixels. Changing alpha by 1/256 is invisible to any viewer.
  • Palette: Reorders the color table of GIF/indexed PNG images to encode bits. Very low capacity, but completely undetectable.
🎵

Audio Methods

  • Audio LSB: Modifies the last bit of each audio sample. At 16-bit audio, the distortion is 1/65536 of the amplitude range — completely inaudible. Great for WAV/FLAC.
  • Phase Coding: Encodes data as phase shifts in audio frequency segments. Survives MP3 compression better than LSB because phase is partially preserved.
  • Spectrogram Art: The most impressive method — embeds a visible image or text in the audio's frequency spectrum. Sounds normal but looks like art in Audacity.
📄

Document Methods

  • Unicode (TXT): Inserts invisible zero-width Unicode characters (U+200B, U+200C, U+200D) between words. The text looks 100% identical — even copy-paste works.
  • PDF Injection: Adds a hidden /StegoData key to the PDF's metadata stream. Invisible in any PDF viewer, but StegoForge can extract it.
  • DOCX/XLSX XML: Adds a custom XML file inside the document's ZIP archive. Word/Excel open the document normally; the hidden file is invisible.
🔐

Encryption & Security

  • AES-256-GCM: Military-grade authenticated encryption. Decrypting with the wrong key returns nothing — not garbage, not an error, nothing.
  • Argon2id KDF: Your passphrase is never used directly. It's hashed with Argon2id (memory-hard, brute-force resistant) to derive the actual key.
  • Decoy Mode: Stores two encrypted payloads. Under pressure, reveal the decoy key (returns innocent content). The real key stays secret.
  • Polymorphic Embed: Your key also determines the pixel traversal order and channel selection, making signature-based detection much harder.
🕵️

Advanced Forensics

  • ML Analysis: Uses neural networks trained on Stego databases to detect probabilistic payload noise.
  • PRNU Fingerprint: Analyzes camera sensor noise. Traditional embedding damages this fingerprint. StegoForge's Fingerprint method preserves it.
  • Chi-Square & RS: Detects flat LSB embedding. "Adaptive LSB" dodges these successfully.
  • Blind Extractor: Brute-forces every parameter combination to attempt payload extraction automatically during CTF mode.
🧪

Survive & Capacity

  • Platform Survival: Simulates social media compression algorithms (Twitter, Discord, etc.) locally using Wet Paper Codes to ensure your payload survives upload.
  • Live Capacity: Dynamic calculation matrix that predicts how much data a carrier can hold across all methods based on its dimensions and depth.
🧬

Hybrid & Experimental

  • Adaptive LSB: Content-aware algorithm that hides data in high-texture, noisy regions to evade detectors.
  • Video Motion: Evaluates temporal cost to embed data inside high-motion video frames, leaving static scenes untouched.
  • Linguistic Text: Context-preserving word substitutions via statistical N-gram analysis to encode payload bits into paragraphs.
  • Decoy Mode: Plausible deniability. Embed two payloads using two keys in the same carrier. Reveal the decoy key under duress.
💻

Using the CLI

StegoForge has a powerful command-line interface. Run stegoforge alone for the interactive menu, or use direct args for scripting:

stegoforge encode -c photo.png -p secret.txt -k "passphrase"
stegoforge decode -f photo_stego.png -k "passphrase"
stegoforge ctf -f challenge.png
stegoforge ctf -f challenge.png --json > report.json
cat file.png | stegoforge ctf --stdin

CTF Tips

  • Always run CTF Mode first on an unknown file — one click, all detectors.
  • Check the confidence score: >70% = very likely stego; <30%=probably clean.
  • If the blind extractor finds candidates, the extracted .bin file may contain the flag directly or be an encrypted blob (look for SFRG header).
  • For EXIF hints: some CTF challenges inject the key or method hint into metadata comments.
  • Try both the JSON output (for scripting) and the visual web UI for different workflows.
